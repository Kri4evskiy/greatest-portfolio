# Code Simplifier

## Principles

- не менять поведение;
- предпочитать ясность компактности;
- уменьшать вложенность и лишние абстракции;
- не тащить новые слои без необходимости;
- не использовать nested ternary там, где читаемость падает.

## Apply when

- код стал перегруженным;
- нужна локальная чистка;
- нужна выверенная небольшая реорганизация без смены логики.
