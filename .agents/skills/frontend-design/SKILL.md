# Frontend Design

## Purpose

Собирать сильный интерфейс, который выглядит как осознанный продукт, а не как шаблонный AI-output.

## Rules

- сначала выбрать визуальное направление;
- типографика и композиция важнее декоративных эффектов;
- не использовать безопасные шаблонные hero/block patterns по инерции;
- не перегружать интерфейс карточками, градиентами и glow-эффектами;
- responsive и accessibility обязательны;
- дизайн должен быть совместим с production-разработкой.

## For this project

- сохранять ощущение современного инженерного продукта;
- уважать контент: статьи, проекты, идеи должны читаться легко;
- визуальный язык должен поддерживать личный бренд, а не спорить с ним.
