# Executing Plans

## Purpose

Выполнять уже утверждённый план без потери контроля над качеством.

## Workflow

1. Прочитать план и проверить, нет ли противоречий.
2. Разбить исполнение на небольшие батчи.
3. После каждого батча запускать релевантные проверки.
4. Фиксировать прогресс кратко и предметно.
5. Останавливаться при блокере, а не угадывать.

## Rules

- не начинать реализацию, если план ещё спорный;
- не пропускать верификацию;
- не расширять scope самовольно;
- при изменении архитектурного решения сначала обновлять документы.
